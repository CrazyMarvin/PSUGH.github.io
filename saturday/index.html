Über den PowerShell Saturday in Hannover

Der PowerShell Saturday findet bei der Firma Elanity Network Partner GmbH in Kooperation mit der PowerShell Usergroup Hannover und mit dem Wissenschatz erfahrener PowerSheller statt.

Denn PowerShell Profis aus allen Bereichen, vom System- bis zum Datenbankadministrator, vom Entwickler bis zum Technischen Evangelisten bei Microsoft werden uns dabei mit Vorträgen und Kenntnissen unterstützen. Dazu gibt es kostenfrei Essen und Getränke.

Wir freuen uns, wenn du bei diesem außergewöhnlichen Event dabei bist. Die Teilnemerzahl ist auf 40 begrenzt. Melde dich deshalb am besten gleich über unsere Meetup-Seite an. Die Teilnahme ist kostenfrei.

Warum man dieses Event auf keinen Fall verpassen darf?

  • Der PowerShell Saturday wird ein großer Spaß und eine gute Gelegenheit, um mit Menschen in Kontakt zu kommen, die von der PowerShell so begeistert sind, dass sie sogar einen Samstag dafür "opfern".
  • Es wird eine Menge neuer Sachen zum Lernen geben, die man gleich ausprobieren kann.
  • Dies ist eine einmalige Chance, sein persönliches Netzwerk zu erweitern, neue Kontakte zu knüpfen, und großartigen Support für die Windows PowerShell zu erhalten.

Speaker Lineup und (vorläufige) Agenda PowerShell Saturday Hannover

10:00 – 10:15 – Begrüßung

10:15 – 11:00 – Einführung in .Net Core und PowerShell 6 mit Christian Imhorst

PowerShell 6 ist eine neue plattformübergreifende (Windows, macOS und Linux) Open Source-Edition von PowerShell, die für heterogene Umgebungen und die Cloud entwickelt wurde und auf das plattformübergreifende .Net Core beruht. Die Frage ist, was bringt uns .Net Core und was sind die Unterschiede von PowerShell 5 zur Version 6?

11:00 – 11:15 – Pause

11:15 – 12:00 – State of the Union: PwSh, rModule, SSH mit Thorsten Butz

PowerShell Core und OpenSSH bringen Windows und Linux-Administratoren auf unerwartete Weise zueinander. Entfaltet sich aber jenseits „nerdiger“ Neugier ein echter Nutzen bei der Administration hetereogener Netzwerke mit der PowerShell Core (PwSh)? Und welches Interesse könnte ein X-affiner DevOp an Microsofts Technologie entwickeln? Wir werfen einen praktischen Blick auf die aktuelle PwSh, das Remoting via SSH, die Funktionalität des "rModule" (Windows PowerShell Compatibility Pack) und gehen der Frage nach, ob die PwSh von den Linux-Distributoren wahrgenommen wird.

12:00 – 12:15 – Pause

12:15 – 13:00 – PowerShell Script Block Logging mit Tobias Weltner

Mit der PowerShell 5 wird mit PowerShell Script Block Logging eine zusätzliche Logging-Funktion eingeführt. Ist das Logging aktiviert, werden alle durch PowerShell ausgeführten Skriptblöcke aufgezeichnet. Wird dann in einem Skriptblock dynamischer Code benutzt, wird neben diesem Code, auch der generierte und tatsächlich ausgeführte Code protokolliert. Dadurch kann man den Code von Anwendungen protokollieren, die bewusst dynamische Codegenerierung zur Verschleierung von Kodierungs- oder Verschlüsselungsmethoden einsetzen, was eine verbreitete Methode zur Umgehung von Sicherheitslösungen unter Windows ist.

13:00 – 14:00 – Mittagessen

14:00 – 14:15 – Was zu Azure mit Jan-Henrik Damaschke

Bei Microsoft Azure handelt es sich um Clouddienste, die stetig erweitert werden, um Ihre Organisation bei geschäftlichen Herausforderungen zu unterstützen. Sie können Anwendungen mithilfe Ihrer bevorzugten Tools und Frameworks in einem großen globalen Netzwerk erstellen, verwalten und bereitstellen.

14:15 – 14:30 – Pause

14:30 – 15:15 – Zertifikate verwalten in PowerShell mit Holger Voges

Zertifikate und Public Private Key Verschlüsselung setzen eine PKI voraus, also einen Zertifikatsserver - oder? Für Testumgebungen geht es auch einfacher, und zwar mit Powershell. Nach einer kurzen Einführung in die Funktionsweise der asymmetrischen Verschlüsselung wird gezeigt, wie man mit Powershell für Testzwecke eine eigene Zertifikatsinfrastruktur baut, wie man mit Powershell Zertifikate verwalten kann und natürlich, wie man seine Daten mit Powershell verschlüsselt.

15:15 – 15:45 – Kaffee und Kuchen

15:45 – 16:30 – Einführung in Pester mit Hands-On mit Marco Kleinert

Pester ist das Testing-Framework für PowerShell und es ist das "must-have" Tool zum Testen von Code und der kompletten IT-Infrastruktur. Alles, was Pester am Ende des Tages tut, ist einen Wert auf der linken Seite (dem aktuellen Zustand) mit einem Wert auf der rechten Seite (dem gewünschten Zustand) zu vergleichen und einen Alarm auszulösen, wenn diese nicht übereinstimmen.Die Verwendung von Pester zum Testen des Codees und der Infrastruktur sollte für IT-Administratoren und Entwickler allgemein üblich werden.

16:30 – 16:45 – Pause

16:45 – 17:30 – Der PSConfTris Automat mit Andreas Nick

Ein in PowerShell geschriebenes Tetris als Modul und als Spielekonsole.
