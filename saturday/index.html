Über den PowerShell Saturday in Hannover

Der PowerShell Saturday findet bei der Firma Elanity Network Partner GmbH in Kooperation mit der PowerShell Usergroup Hannover und mit dem Wissenschatz erfahrener PowerSheller statt.

Denn PowerShell Profis aus allen Bereichen, vom System- bis zum Datenbankadministrator, vom Entwickler bis zum Technischen Evangelisten bei Microsoft werden uns dabei mit Vorträgen und Kenntnissen unterstützen. Dazu gibt es kostenfrei Essen und Getränke.

Wir freuen uns, wenn du bei diesem außergewöhnlichen Event dabei bist. Die Teilnemerzahl ist auf 40 begrenzt. Die Teilnahme ist kostenfrei.

Warum man dieses Event auf keinen Fall verpassen darf?

  • Der PowerShell Saturday wird ein großer Spaß und eine gute Gelegenheit, um mit Menschen in Kontakt zu kommen, die von der PowerShell so begeistert sind, dass sie sogar einen Samstag dafür "opfern".
  • Es wird eine Menge neuer Sachen zum Lernen geben, die man gleich ausprobieren kann.
  • Dies ist eine einmalige Chance, sein persönliches Netzwerk zu erweitern, neue Kontakte zu knüpfen, und großartigen Support für die Windows PowerShell zu erhalten.

15:45 – 16:30 – Das PackageManagement-Modul mit Patrik Horn & Andreas Nick

PackageManagement ist ein Windows 10 internes PowerShell Modul , welches das Arbeiten mit unterschiedlichen Installer-Technologien ermöglicht. Nicht nur die Verwaltung bekannter Formate wie MSI/MSU Pakete ist interessant. Sondern auch die Erstellung eigner Paketprovider zur Installation von Anwendungen. Diese Session wird dieses Modul und die Möglichkeiten dieses Modules vorstellen. Weiterhin, wie man selber einen minimalistischen PackageProvider erstellen kann, beispielsweise zur Verwaltung von Installationen (und Skripten) in einem Verzeichnis.

Track 2

10:15 – 13:00 – Powershell-Einführung

mit Holger Voges
Die PowerShell ist eine Befehlszeilenshell, die mit Blick auf Systemadministratoren entwickelt wurde. Seit Windows 7 bzw. Windows Server 2008 R2 ist die Powershell direkt ins Betriebssystem integriert. PowerShell umfasst dabei eine interaktive Eingabeaufforderung und eine Skriptumgebung, was beides kombiniert werden kann. Anders als die meisten Shells, in denen Text zurückgegeben wird, setzt PowerShell auf .NET Objekte. Durch diese grundlegende Änderung in der Umgebung hat man als Admin völlig neue Tools und Methoden für die Verwaltung und Konfiguration von Windows.
